# Energy8 Build Deploy System v2.1

🚀 **Продвинутая система управления сборками Unity с автоматической интеграцией Unity Build Profiles**

Автоматически сканирует ваши Build Profiles и создает мощные надстройки для развертывания с GUID-based привязкой к ассетам для корректного переименования.

## 🎯 Плейсхолдеры в Output Path

В поле **Output Path** можно использовать плейсхолдеры, которые автоматически заменяются при сборке:

- `{{{VERSION}}}` - Версия проекта (например: `1.2.3`)
- `{{{PLATFORM}}}` - Целевая платформа (`Windows`, `WebGL`, `Android`, `iOS`, `macOS`, `Linux`)  
- `{{{ENVIRONMENT}}}` - Текущая среда (Development, Debug, Production и т.д.)
- `{{{DATE}}}` - Текущая дата (`yyyy-MM-dd`)
- `{{{DATETIME}}}` - Дата и время (`yyyy-MM-dd_HH-mm`)

**Примеры:**
```
Builds/{{{PLATFORM}}}/{{{VERSION}}}                →  Builds/WebGL/1.2.3
Builds/{{{PLATFORM}}}/{{{ENVIRONMENT}}}/{{{VERSION}}}  →  Builds/WebGL/Production/1.2.3
Releases/{{{DATE}}}_{{{PLATFORM}}}_v{{{VERSION}}}      →  Releases/2025-06-22_Windows_v1.2.3
Output/{{{PLATFORM}}}/Build_{{{DATETIME}}}       →  Output/Android/Build_2025-06-22_14-30
```

## 🌍 Управление средами (Environment Management)

Система интегрирована с пакетом **io.energy8.environment-config** для управления конфигурациями сред при сборке.

### Настройка среды в конфигурации сборки

В каждой конфигурации сборки можно задать целевую среду:
- **Поле "Target Environment"**: Укажите название среды (Development, Debug, Production и т.д.)
- **Если поле пустое**: Будет использоваться текущая среда из `environment.json`

### Автоматическое переключение среды

Перед началом сборки система автоматически:
1. Проверяет заданную `Target Environment` в конфигурации
2. Если среда задана - обновляет `environment.json` в `StreamingAssets/E8Config/`
3. Логирует изменение среды: `"[BuildSystem] Environment switched to: Production"`

### Доступные среды

Если установлен пакет **io.energy8.environment-config**, в UI отображаются:
- Список всех доступных сред из `E8EnvironmentSettings`
- Цветовая маркировка каждой среды
- Кнопка для быстрого переключения на среду

### Примеры использования

**Пример 1: Сборка для конкретной среды**
```
Output Path: Builds/{{{PLATFORM}}}/{{{ENVIRONMENT}}}/{{{VERSION}}}
Target Environment: Production
Результат: Builds/WebGL/Production/1.2.3
```

**Пример 2: Использование текущей среды**
```
Output Path: Builds/{{{PLATFORM}}}/{{{VERSION}}}
Target Environment: (пусто)
Результат: Builds/WebGL/1.2.3 (используется текущая среда из environment.json)
```

### Интеграция с Environment Config

Для полноценной работы управления средами:
1. Установите пакет **io.energy8.environment-config**
2. Настройте среды через `E8 Tools → Environment Switcher`
3. В Build Deploy System выберите нужную среду в каждой конфигурации
4. Система автоматически переключит среду перед сборкой

> 💡 **Совет**: Создайте отдельные конфигурации сборки для каждой среды (Development, Debug, Production) для удобного переключения.

**Как работает:**
1. Выбираете нужные форматы в настройках WebGL
2. Система автоматически собирает основной билд + дополнительные форматы
3. Копирует `.data` файлы с префиксами (`_astc`, `_etc2`) в основную папку
4. Template `index.html` автоматически выбирает подходящий формат для устройства

### 🎛️ Платформо-специфичные настройки
- **WebGL**: Форматы текстур, компрессия, кэширование
- **Android**: AAB/APK, keystore, архитектуры ARM64/ARMv7
- **iOS**: Team ID, подписание, simulator/device builds

## 🎯 Ключевые особенности v2.0

### 🔄 Автоматическая синхронизация
- **Автосканирование**: Мониторит папку `Assets/Settings/Build Profiles`
- **Реалтайм обновления**: AssetPostprocessor отслеживает изменения
- **GUID привязка**: Устойчивые ссылки на Build Profile ассеты
- **Автосоздание конфигов**: Генерирует надстройки в `Assets/BuildSystem/Configs`

### 📦 Умное управление конфигурациями
- **ScriptableObject архитектура**: Каждая конфигурация - отдельный .asset файл
- **Переименование friendly**: GUID-based ссылки не ломаются при переименовании
- **Автоочистка**: Удаляет конфигурации для несуществующих профилей
- **Валидация в реалтайме**: Мгновенная проверка корректности настроек

### 🛠️ Расширенные возможности сборки
- **ZIP архивирование**: Автоматическое создание архивов сборок
- **Clean Before Build**: Очистка папки перед сборкой
- **Custom Build Names**: Переопределение имен исполняемых файлов
- **Auto Path Generation**: Умная генерация путей на основе профилей

## � Как работает система

1. **Сканирование**: При запуске Unity автоматически сканирует `Assets/Settings/Build Profiles`
2. **Создание конфигов**: Для каждого Build Profile создается конфигурация в `Assets/BuildSystem/Configs`
3. **Привязка по GUID**: Конфигурации ссылаются на профили через GUID для устойчивости
4. **Мониторинг изменений**: AssetPostprocessor отслеживает добавление/удаление профилей
5. **Автообновление**: Конфигурации синхронизируются с изменениями автоматически

## � Быстрый старт

### 1. Откройте окно системы
```
Unity Menu → Energy8 → Build Deploy System
```

### 2. Автоматическая настройка
- Система автоматически найдет ваши Build Profiles
- Создаст конфигурации в `Assets/BuildSystem/Configs`
- Если конфигурации не появились, нажмите "🔄 Обновить конфигурации"

### 3. Настройте конфигурации
- **Custom Name**: Переопределите имя сборки
- **Output Path**: Укажите папку для вывода
- **Auto Generate Path**: Автоматическая генерация путей
- **Clean Before Build**: Очистка перед сборкой
- **Create ZIP Archive**: Создание архива после сборки

### 4. Выполните сборку
- Выберите конфигурацию (радиокнопка)
- Нажмите "🔨 Build" или "🧹 Clean Build"

## 🏗️ Архитектура системы

```
Assets/
├── Settings/
│   └── Build Profiles/          # Unity Build Profiles (исходные)
│       ├── Assets Debug.asset
│       ├── Assets Dev.asset
│       └── Lite.asset
└── BuildSystem/
    └── Configs/                 # Конфигурации надстроек (создаются автоматически)
        ├── Config_Assets_Debug.asset
        ├── Config_Assets_Dev.asset
        └── Config_Lite.asset
```

### Компоненты системы

#### � BuildProfileScanner
- Сканирует Build Profiles
- Создает и обновляет конфигурации
- Очищает устаревшие конфигурации

#### �️ BuildProfileWatcher  
- AssetPostprocessor для мониторинга изменений
- Автоматически вызывает обновление при изменениях

#### ⚙️ BuildConfiguration (ScriptableObject)
- Привязка к Build Profile через GUID
- Настройки развертывания
- Автоматическая валидация

#### �️ BuildDeployWindow
- Современный UI с эмодзи
- Реалтайм валидация
- Интуитивное управление конфигурациями

#### � BuildManager
- Выполнение сборок
- ZIP архивирование
- Система событий для прогресса

## 🎮 Примеры использования

### Пример 1: Автоматическая настройка
```csharp
// Система автоматически создаст конфигурации для всех профилей
var configs = BuildProfileScanner.ScanAndCreateConfigurations();
Debug.Log($"Создано {configs.Count} конфигураций");
```

### Пример 2: Программная сборка
```csharp
// Найти конфигурацию по имени профиля
var configs = BuildProfileScanner.GetAllConfigurations();
var config = configs.FirstOrDefault(c => c.BuildProfileName == "Assets Debug");

if (config != null && config.IsValid())
{
    BuildManager.BuildConfiguration(config);
}
```

- **Unity версия**: 2022.3+
- **Поддерживаемые платформы**: Windows, macOS, Linux, WebGL, Android, iOS
- **Архитектура**: GUID-based ссылки, ScriptableObject конфигурации
- **Автообновление**: AssetPostprocessor + InitializeOnLoad
- **Зависимости**: Только стандартные Unity API

### 📋 Требования для WebGL компрессии
Для использования компрессии WebGL файлов необходимо установить в системе:
- **gzip.exe** - для GZIP компрессии
- **brotli.exe** - для Brotli компрессии

Утилиты должны быть доступны через PATH системы. Система автоматически найдет их и использует для сжатия .data, .wasm и других файлов WebGL сборки.

## 🐛 Решение проблем

### Конфигурации не создаются
1. Проверьте наличие Build Profiles в `Assets/Settings/Build Profiles`
2. Нажмите "🔄 Обновить конфигурации"
3. Проверьте консоль на ошибки

### Build Profile не найден
- Конфигурация показывает ⚠️ - профиль был удален или перемещен
- Система автоматически удалит устаревшие конфигурации при следующем сканировании

### Права доступа к папке
- Убедитесь что Unity имеет права записи в папку `Assets/BuildSystem/Configs`

### Компрессия WebGL не работает
- Убедитесь что `gzip.exe` и `brotli.exe` установлены в системе
- Проверьте что утилиты доступны через PATH: откройте командную строку и выполните `gzip --version` и `brotli --version`
- В Windows можно установить через chocolatey: `choco install gzip brotli`
- В macOS через homebrew: `brew install gzip brotli`
- В Linux через пакетный менеджер: `apt install gzip brotli` или `yum install gzip brotli`
